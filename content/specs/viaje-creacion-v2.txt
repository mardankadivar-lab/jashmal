Quiero rediseñar la página de Jashmal Creation como una experiencia inmersiva de scrollytelling cabalístico sobre el proceso de la Creación según Etz Chaim.

URL actual:
https://jashmal.org/en/creacion

Objetivo:
Transformar la página en una experiencia visual, espiritual e interactiva. No debe sentirse como un artículo ni como una página educativa común. Debe sentirse como entrar dentro del proceso secreto de la Creación: Ein Sof, Tzimtzum, Chalal Panui, Kav, Adam Kadmon, Shevirat HaKelim, Nitzotzot, Tikkun y los Cuatro Mundos.

La experiencia debe ser tipo Apple scrollytelling:
- Scroll vertical largo, suave e inmersivo.
- El elemento central permanece fijo/pinned mientras el usuario hace scroll.
- Cada sección transforma visualmente la escena anterior.
- La creación debe sentirse como una sola continuidad, no como slides separados.
- El usuario debe sentir que desciende dentro de la luz, el vacío, las vasijas, las chispas y los mundos.

IMPORTANTE:
No quiero una página estática.
No quiero tarjetas comunes.
No quiero bloques largos de texto.
Quiero una experiencia cinematográfica interactiva.

TECNOLOGÍA RECOMENDADA:

Frontend:
- Next.js 14+ con App Router.
- React.
- TypeScript.
- Tailwind CSS.
- Framer Motion para animaciones de UI, textos, fades y transiciones.
- GSAP + ScrollTrigger para controlar el scroll, secciones pinned y transformaciones precisas.
- Lenis para smooth scrolling premium.
- React Three Fiber / Three.js para escenas 3D, partículas, luz, esferas, cosmos y profundidad.
- Drei para helpers de Three.js.
- Zustand para controlar el estado activo de la escena.
- SVG para símbolos hebreos, líneas, círculos y geometría sagrada.
- Rive o Lottie opcional para microanimaciones exportadas.
- Dynamic imports para cargar Three.js solo en cliente.
- prefers-reduced-motion para accesibilidad.

Arquitectura visual:
Crear una experiencia de scroll continuo con una escena central fija.

Estructura técnica:
- Componente principal: CreationExperience.tsx
- Componente de escena 3D: CreationCanvas.tsx
- Componente de texto flotante: SceneCopy.tsx
- Componente de navegación lateral: SceneProgress.tsx
- Componente de panel profundo: StudyPanel.tsx
- Data source: creationScenes.ts

EXPERIENCIA DE SCROLL:

El scroll debe sentirse casi infinito, como una inmersión continua.

Implementación:
- Usar Lenis para smooth scroll.
- Usar GSAP ScrollTrigger para crear un contenedor pinned de gran altura.
- La escena 3D permanece fija en el centro.
- El progreso del scroll controla el estado visual.
- Cada escena ocupa aproximadamente 100vh o 150vh.
- Las transiciones entre escenas deben ser graduales.
- Al final, la experiencia puede hacer una transición circular: las chispas del Tikkun vuelven a formar una luz superior, conectando nuevamente con Ein Sof. Esto da sensación de ciclo infinito.

Ejemplo conceptual:
Ein Sof → Luz infinita → Contracción → Vacío → Huella → Línea → Vasijas → Ruptura → Chispas → Reparación → Mundos → Alma humana → Retorno a la Luz.

VISUALES POR ESCENA:

1. Ein Sof
Pantalla casi totalmente negra.
No mostrar estrellas todavía.
Solo una respiración sutil de luz invisible.
Texto:
אֵין סוֹף
Before form, before time, before worlds —
there was only the Infinite.

2. Or Ein Sof
La pantalla se llena de luz blanca/dorada.
No hay objetos. Solo luz.
Texto:
The light did not illuminate something.
The light was everything.

3. Tzimtzum
La luz se retrae desde el centro.
Aparece un círculo oscuro.
Texto:
For a world to exist,
the light concealed itself.

4. Chalal Panui
Vacío circular profundo.
Luz infinita alrededor.
Texto:
The void was not absence.
It was possibility.

5. Reshimu
Dentro del vacío queda polvo dorado, una memoria de luz.
Texto:
Even after concealment,
a trace remained.

6. Kav
Un rayo vertical entra desde arriba hacia el vacío.
Texto:
A single line entered the empty space —
infinite light measured into form.

7. Adam Kadmon
La línea empieza a formar una estructura humana abstracta hecha de luz, geometría y sefirot.
No debe parecer un cuerpo literal.
Texto:
The first configuration of light
was called Adam Kadmon.

8. Akudim
Luces múltiples atadas dentro de una sola vasija.
Texto:
Many lights.
One vessel.

9. Nekudim
Puntos de luz separados, intensos e inestables.
Texto:
The light divided into points.
Each point carried more than it could contain.

10. Shevirat HaKelim
Clímax visual.
Vasijas luminosas se fracturan.
Chispas caen hacia abajo.
Texto:
The vessels shattered.
The light descended in fragments.

11. Nitzotzot
Miles de chispas atrapadas en oscuridad.
Texto:
Every fragment hid a spark.
Every spark waits to be lifted.

12. Tikkun
Las chispas empiezan a reorganizarse.
Aparecen líneas, armonía, reparación.
Texto:
Creation does not end in rupture.
It begins in repair.

13. Four Worlds
Cuatro capas descendentes:
Atzilut
Beriah
Yetzirah
Asiyah

Visual:
Cuatro planos transparentes de luz descendiendo.
Texto:
The light descended through worlds
until it touched matter.

14. Human Participation
Una silueta humana abstracta, o una Alef formada por chispas.
Texto:
The human being appears not as an accident,
but as a partner in Tikkun.

Final:
The world was not created finished.
It was created to be repaired.

- Cada escena representa un rango de progreso.
- Al llegar al final, las partículas del Tikkun deben reorganizarse en una luz circular que recuerda al Ein Sof.
- Opcional: agregar botón “Return to Ein Sof” que hace smooth scroll al inicio.
- Opcional: crear un pequeño loop visual donde las chispas finales vuelven a convertirse en luz superior, para sugerir que la creación es un proceso cíclico de ocultamiento y revelación.

PERFORMANCE:

Optimizar para mobile y desktop:
- Limitar número de partículas en móvil.
- Usar devicePixelRatio controlado en Three.js.
- Lazy load del canvas.
- Pausar animaciones fuera de viewport.
- Usar CSS transform y opacity para animaciones simples.
- Mantener el texto como HTML accesible, no dentro del canvas.

RESPONSIVE:

Desktop:
- Escena central grande.
- Texto a la izquierda o debajo según la escena.
- Navegación lateral derecha.

Mobile:
- Escena central ocupa la parte superior/media.
- Texto aparece abajo.
- Navegación lateral reducida o convertida en progress bar inferior.
- Botón “Study in depth” visible pero discreto.
